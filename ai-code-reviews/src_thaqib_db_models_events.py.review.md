# Review: src/thaqib/db/models/events.py

Overall quality: Good

Refactoring effort: Medium

## Findings

1. Event and alert statuses are raw strings
   - Lines `68`, `70`, `119`, and `122` use strings for event type/severity/alert type/status.
   - Suggestion: introduce enums or check constraints for all state-machine fields.

2. Media paths are stored as strings
   - Lines `78-80` store video/audio clip paths.
   - Security impact: path handling must remain carefully constrained in serving routes.
   - Suggestion: store media object IDs or normalized relative paths generated by a media service.

3. JSON fields lack shape constraints
   - Lines `38`, `72`, and `82` store JSON blobs.
   - Suggestion: define schema versions in metadata and validate event payload shapes before persistence.

## What Works

- Event relationships to exam session, device, group event, and alerts are explicit.
- Cascades on session deletion are at least intentionally configured.
